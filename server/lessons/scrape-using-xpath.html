III. Scrape to monitor content on a webpage

Scrape a section of a website on a schedule and receive alerts when its content changes

Overview

In this tutorial we will use the City's auditor Office website of the City of Atlanta. Our goal is to be notified everytime a new audit report is published.

Scrape the HTML from the webpage

First, we need to scrape the HTML text of the webpage containing the data we want to work with. That HTML contains all the information we need. We will store it securely.

Instructions

  1. Add the Scrape 'HTML scraper'.
  2. Paste in the following URL, where all the public reports are listed, and press Scrape
    http://www.atlaudit.org/audit-reports.html

    Next, let's extract our data.

Extract data from the HTML

Now, we need to extract specific data from the HTML we just saved, and store it in a new table so we can work with it.

To do this, we will use the XPath extractor.

Instructions

  1. Add the 'Xpath extractor' step.

Get the XPath selector to extract the list

Xpath is a language for selecting elements within HTML documents. It’s called a “path” because it identifies a particular element by specifying its parent element, then the parent’s parents, all the way to the root of the HTML tree.

We recommend using the SelectorGadget Chrome browser extension. It is a point-and-click interface for visually selecting elements on the page and generating Xpath selectors.

Editing the Xpath

Now that we have a list of all documents, let's extract the links of each of them. To do so, let's first have a look at the structure of our page's HTML to understand how this all works.

Naviguate back to the webpage.

In that tab, scroll down to list of all publications released by the City Auditor's Office. Right-click on the first link in that list, and choose Inspect in the menu to open the Inspector panel.

Let's look closely at the HTML element in the browser's inspector. In the Inspector panel, the <a> element corresponding to the link you are inspecting is highlighted.

Xpath is a language for selecting elements within HTML documents. It’s called a “path” because it identifies a particular element by specifying its parent element, then the parent’s parent, all the way to the root of the HTML tree.

In the inspector, we can see the <a> tag we selected is nested within a <li> parent, which is itself nested within a <ul>. Now look closely at the XPath selector generated with Selector Gadget:

//*[(@id = "wsite-content")]//li//a
  • //*[(@id = "wsite-content")] means "Within the page, select all elements with the id wsite-content."
  • //li means "Within it, select all li elements".
  • The last //a means "Within it, select all a elements". Those elements are links.

In order to get the URL of the link, and not just the element, we need to append one more section to the code, which is visible in your inspector but not detected by Selector Gadget: /@href.

Extracting a second column

Let's extract the URLs associated with each links and store them in a new column.

Instructions

  1. Use the +ADD button to create a new column. Copy and paste the Xpath you entered in the first Column selector, paste it into the new one, and append /@href at the end of it in order to target the URL. Name this new column URL, and validate your settings

Set up the alert

Let's set up the scraper to monitor the webpage and automatically get new data when it is published.

Instructions

  1. In the HTML scraper, click manual to open the Workflow Update window. Set the feature to Auto. Click Apply.
  2. In the Xpath extractor, click the alert icon and select Turn on to receive an email if the output of this step changes.

Your scrapers are set!

If the new data is published, new tables will be extracted and you will receive an email!
