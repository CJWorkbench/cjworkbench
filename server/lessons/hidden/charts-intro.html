Column and line charts

Introduction

Not all data stories need a data visualization. If you have only one or two numbers, they can probably just appear as text in the final story.
But with more data, charts are essential to quickly convey large amounts of information.

In this lesson we will work with a prepared version of the same dataset, containing the 10 largest C02 emitters since 2010.

Tranpose 1

Let’s look closely at our table.
In Workbench charts are plotted by selecting data in columns. We cannot select China, because it is displayed as a row.
Likewise, there is no column Year to select. Instead, each year is its own column.

If we want to select variables that are displayed as rows, we must turn them into columns. This is called transposing.

Tranpose 2

Transposing turns rows into columns and columns into rows. Doing so, it swaps the header with the left-most column.

Since the header does not have a name, we can name it when it becomes the new left-most column. Likewise, after transposing, the name of the left-most column will not be displayed anymore.

Transpose 3

    Now we can plot China’s output.

  3. Add a Column Chart, select Year as the X axis, and China as the Y axis.

When to use a line chart

If column charts are a good choice to visually compare how much individual teams each spend in one season, a line chart is more adapted to visually express trends through time. The reason is that continuity can be established between points in time, and so a line can be drawn to help viewers grasp the direction of the change.
Line charts, however, should not be used with anything else than a horizontal axis representing time. Drawing a line between ’China’ and ‘Russia’, would make no sense: There is no in-between value, and the chart would show something that doesn’t exist!
