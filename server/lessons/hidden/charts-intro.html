Column and line charts

Introduction

Not all data stories need a data visualization. If you have only one or two numbers, they can probably just appear as text in the final story. With more data, charts are essential to quickly convey large amounts of information to your audience.

In this lesson we will work with the same dataset we used in the first lesson, containing international data on the 10 largest C02 emitters since 2010.

When to transpose

Let's visualy compare the amount of CO2 emitted by China to that of other countries.

Let’s look closely at our table.
In Workbench charts are plotted by selecting data in columns.

We cannot select China, because it is displayed as a row. Likewise, there is no column Year to select. Instead, each year is its own column.

If we want to select variables that are displayed as rows, we must turn them into columns. This is called transposing.

How Transpose works

Transposing swaps the header with the left-most column. Doing so, it turns rows into columns and columns into rows.

Since the left-most column in fig.1 becomes a header in fig.2, you will often have to rename the new left-most column in fig.2.

Transpose in action

When to use a line chart

If column charts are a good choice to visually compare how much C02 is produced by individual countries in a given time frame, a line chart is more adapted to visually express trends through time. The reason is that continuity can be established between points in time, and so a line can be drawn to help viewers grasp the direction of the change.

Line charts, however, should not be used with anything else than a horizontal axis representing time. Drawing a line between ’China’ and ‘Russia’, would make no sense: There is no in-between value, and the chart would show something that doesn’t exist!
