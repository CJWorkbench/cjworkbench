Tranposing

Let’s look closely at our table.
We cannot select China, because it is displayed as a row. We need it to be a column. Likewise, there is no column Year to select. Instead, each year is its own column.

If we want to access variables that are displayed as rows, we must turn them into columns, which is called transposing.

Transposing

Transposing turns rows into columns and columns into rows. Doing so, it swaps the header with the left-most column. Since the header does not have a name, we must name it when it becomes the new left-most column. Likewise, after transposing, the name of the left-most column will not be displayed anymore.

When to use a line chart

If column charts are a good choice to visually compare how much individual teams each spend in one season, a line chart is more adapted to visually express trends through time. The reason is that continuity can be established between points in time, and so a line can be drawn to help viewers grasp the direction of the change.
Line charts, however, should not be used with anything else than a horizontal axis representing time. Drawing a line between ’China’ and ‘Russia’, would make no sense: There is no in-between value, and the chart would show something that doesn’t exist!
