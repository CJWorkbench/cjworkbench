Long and wide format

Introduction

Data can be organized into tables in different ways. In this lesson, you will learn to recognize “long” and “wide” formats and convert between them, which is called reshaping.

Long format have two properties:

There is only one column with measured values.
In this case, “Count of incidents”. All other columns are variables that describe what is measured.

Each row is a now unique combination of those variables.
In this case, each available "Complaint date" is combined with each unique borough.

Wide formats are the commonly found table format. In a wide format, there are one or more columns that identify what each row of data refers to, in this case “Complaint date.” These columns are called “row variables.”
All remaining columns contain measurements. The column names of those columns are the "column variable", in this case each borough.

Converting to wide format

For this data, our goal is to to visualize the volume of incidents reported in each borough, for each week. Because Workbench charts plot columns, each borough must be in its own column, with the rows corresponding to dates.

Instructions

  1. Add ‘Reshape’, and select Long to wide.
  2. Select date as the Row variable.
  3. Select Boroughfor the Column variable.

Visualizing

Finally, we’re ready to visualize our grouped police incident data.

Let’s visually compare the volume of reported police incidents in each borough, for each week, using a line chart.

  1. Add a line chart.
  2. Choose Complaint date for the X axis.
  3. Add one column on the Y axis for each Borough you want to compare.
