Long and wide format

Learn the difference between “long” and “wide” table formats

Introduction

Data can be organized into tables in different ways. In this lesson, you will learn to recognize “long” and “wide” formats and convert from one to another, which is called reshaping.

In this lesson we will work with a month of the same data we used in the previous lesson, on police incident reported in the City of New York.

Long format tables contain only one column with measured values.
In this case Count of incidents. All other columns are variables that describe what is measured.

Each row is a unique combination of those variables.
In this case, each available Complaint date is combined with each unique Borough.

Wide formats are the most common table format. In a wide format. there are one or more columns that describe what is measured, in this case Complaint date. These columns are called “Row variables".
All other column names are called "Column variables", as they also describe what is measured - in this case each Borough. All remaining cells contain the actual measurements.

Converting to wide format

Our goal is to visualize the volume of incidents reported in each borough, for each day.

All dates must be in one column so that we can select it as the X axis, and each borough must be in its own column, so that we can pick and choose which we want to compare.

Instructions

  1. Add ‘Reshape’, and select Long to wide.
  2. Select Complaint date as the Row variable.
  3. Select Boroughfor the Column variable.

Visualizing

Finally, we’re ready to visualize our grouped police incident data.

Let’s visually compare the volume of reported police incidents in each borough, using a line chart.

  1. Add a line chart.
  2. Choose Complaint date for the X axis.
  3. Add one column on the Y axis for each Borough you want to compare.
