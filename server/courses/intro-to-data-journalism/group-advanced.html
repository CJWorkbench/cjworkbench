Grouping by dates, and multiple columns

Learn advanced grouping techniques to summarize data fast and efficiently.

Introduction

In the last lesson we learned how to ensure that a column of dates has the correct “Date” type. Now we’re ready to group by date, using the same police incident data from the City of New York we used in the previous lesson.

We’ll answer the question: how many felonies were reported each month in each borough of the city?

Grouping by date

This table was describes all felonies reported by the police of New York City for the months of January and February 2018. The original is published here with a data dictionary.

Our first question is ‘How many incidents were reported each month, in the entire city?’

Instructions

  1. Add Group, and select the column Complaint date.
  2. Check Group dates and select by month as granularity.
  3. Name the output column Count of incidents.

Granularity

Let’s visualize this data in a line chart.

Instructions

  1. Add a Line chart. Choose Complaint date as the X axis and plot Count of incidents on the Y-axis.

    Charts with two data points are not very useful. To visualize the same data but for each day, we can change the granularity of the group.

  2. In the Group in step 5, change the granularity to by day.

    Select step 6 to display your new chart.

Grouping by multiple columns

Now let’s break down this data further, by borough.

Instructions

  1. In the Group step, add the Borough column next to Complaint date using the same column selector.

    The table now displays a more detailed count of incidents. It shows how many felonies were reported in each Neighborhood, within each day.

    By looking at this table we can rapidly see if different parts of town have different patterns of crime on any given day - or at least reported incidents. It’s important to remember that data only exists when someone types it in.

Lesson Complete!

After being grouped by more than one column the number of incidents per borough cannot be directly visualized, as it would require the data for each borough to be in its own column.
